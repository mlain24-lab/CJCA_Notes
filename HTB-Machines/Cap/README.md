# HTB Write-Up: Cap
**Platform:** Hack The Box
**Certification Path:** Certified Junior Cybersecurity Associate (CJCA)
**Target IP:** 10.129.61.49
**Key Techniques:** Network Enumeration, Traffic Analysis (PCAP), IDOR Exploitation, Password Reuse, Privilege Escalation (Linux Capabilities).

---

## 1. Reconnaissance & Enumeration
- **Port Scanning:** An initial network footprinting scan revealed **3 open TCP ports**: 
  - Port 21 (FTP - Cleartext protocol)
  - Port 22 (SSH)
  - Port 80 (HTTP - Gunicorn/Werkzeug)
- **Web Application Analysis:** The HTTP service hosts a "Security Dashboard" that generates network snapshots. Interacting with the capture feature redirects the user to a specific URI path formatted as `/data/[id]`, which displays the scan metrics.
- **Vulnerability Discovery (IDOR):** By intercepting the requests and manipulating the `[id]` parameter, the application granted unauthorized access to historical network captures generated by other users. This confirmed a critical Insecure Direct Object Reference (IDOR) vulnerability.

## 2. Initial Access & Data Exfiltration
- **Traffic Exfiltration:** Exploiting the IDOR flaw, iterative requests were sent to the `/download/[id]` endpoint to retrieve the actual binary files. The historical PCAP file containing sensitive data was identified under **ID 0**.
- **Cleartext Traffic Analysis:** Deep packet inspection of the downloaded PCAP file using command-line string extraction revealed unencrypted **FTP** traffic. Because FTP does not enforce transport layer security, the authentication sequence was captured, successfully exposing the credentials for the user `nathan`.
- **Password Reuse Exploitation:** Leveraging the concept of credential stuffing and password reuse, the compromised FTP password was tested against the **SSH** service. Authentication was successful, securing an interactive Initial Access shell and yielding the `user.txt` flag.

## 3. Privilege Escalation
- **System Enumeration:** Post-exploitation enumeration focused on identifying kernel-level permission misconfigurations. Instead of standard SUID binaries, a sweep for abnormal Linux Capabilities was conducted using the command: `getcap -r / 2>/dev/null`.
- **Vulnerable Binary:** The enumeration highlighted that the Python interpreter located at `**/usr/bin/python3.8**` had been assigned the `cap_setuid` capability.
- **Root Shell Access:** This misconfiguration allowed the Python binary to arbitrarily alter its effective User ID (UID) without requiring full `sudo` privileges. By executing a one-liner payload (`import os; os.setuid(0); os.system("/bin/bash")`), the process successfully escalated to UID 0, granting a root shell and granting access to the `root.txt` flag.
